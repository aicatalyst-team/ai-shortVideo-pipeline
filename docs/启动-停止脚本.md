## 一、前置检查（所有平台）

### 1.1 软件要求

| 软件 | 最低版本 | 必需？ |
|---|---|---|
| Docker Desktop | 24.0 | ✅ |
| Docker Compose | v2.20（Docker Desktop 自带） | ✅ |
| Git | 2.30 | ✅ |
| Node.js | 20.0 | 仅前端开发 |
| Python | 3.11 | 仅本地直跑 |

### 1.2 `.env` 必须存在

仓库根目录的 `.env`（**不进 git**）由 `.env.example` 拷贝而来。最少必填：

```bash
DEEPSEEK_API_KEY=
GLM_API_KEY=
KLING_ACCESS_KEY=
KLING_SECRET_KEY=
GATEWAY_AUTH_JWT_SECRET=      # ≥ 32 字节强随机
```

生成 JWT secret 的命令：

```bash
# Linux / macOS
openssl rand -base64 32

# Windows PowerShell
[Convert]::ToBase64String((1..32 | ForEach-Object {[byte](Get-Random -Max 256)}))
```

### 1.3 端口检查

| 端口 | 服务 |
|---|---|
| 5173 | 前端 dev server |
| 8000 | FastAPI orchestrator |
| 8080 | Java gateway |
| 5432 | Postgres |
| 6379 | Redis |
| 9000 / 9001 | MinIO API / Console |

冲突时改 `docker-compose.yaml` 中对应服务的 `ports` 段，或先停掉占用进程。

## 二、启动场景选择

| 场景 | 用途 | 起的服务 | 首次耗时 |
|---|---|---|---|
| **A. 前端 Mock** | 改 UI / 截图 / 演示 | sl-vue（前端） | 30 秒 |
| **B. 后端 + 前端** | 真实开发 / 调 webhook | postgres + redis + orchestrator + worker + 前端 | 10-15 分钟 |
| **C. 全栈含 Java 网关** | 测 JWT / failover / 计量 / 全链路 trace | B + gateway | +10-15 分钟（首次 build） |

> 默认情况下推荐 **场景 B**，等需要测网关治理能力时再切换到 **场景 C**。

## 三、Windows 一键启动 / 停止

### 3.1 启动（PowerShell）

```powershell
# 进入仓库根目录
cd path\to\myAiVideos

# 场景 A：仅前端 Mock
cd sl-vue
$env:VITE_USE_MOCK = "true"
npm install
npm run dev

# 场景 B：后端 + 前端（一键）
docker compose up -d postgres redis orchestrator worker
docker compose exec -T orchestrator alembic upgrade head
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd $PWD\sl-vue; npm install; npm run dev"

# 场景 C：B + Java 网关
docker compose up -d postgres redis orchestrator worker gateway
docker compose exec -T orchestrator alembic upgrade head
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd $PWD\sl-vue; npm install; npm run dev"
```

### 3.2 停止（PowerShell）

```powershell
# 停 Docker 服务（数据 volume 保留）
docker compose stop

# 关掉前端 dev server（占用 5173 的 Node 进程）
Get-NetTCPConnection -LocalPort 5173 -ErrorAction SilentlyContinue |
    Select-Object -First 1 -ExpandProperty OwningProcess |
    ForEach-Object { Stop-Process -Id $_ -Force }
```

### 3.3 一键脚本（推荐）

如果项目里 `scripts/start-local.ps1` 已就绪：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start-local.ps1
# 附 -WithGateway 启动场景 C
powershell -ExecutionPolicy Bypass -File scripts\start-local.ps1 -WithGateway
# 附 -NoFrontend 只起后端
powershell -ExecutionPolicy Bypass -File scripts\start-local.ps1 -NoFrontend

# 停止
powershell -ExecutionPolicy Bypass -File scripts\stop-local.ps1
```

## 四、macOS / Linux 一键启动 / 停止

### 4.1 启动（Bash / Zsh）

```bash
# 进入仓库根目录
cd path/to/myAiVideos

# 场景 A：仅前端 Mock
cd sl-vue
VITE_USE_MOCK=true npm install && VITE_USE_MOCK=true npm run dev

# 场景 B：后端 + 前端
docker compose up -d postgres redis orchestrator worker
docker compose exec -T orchestrator alembic upgrade head
(cd sl-vue && npm install && nohup npm run dev > /tmp/sl-vue.log 2>&1 &)
echo $! > /tmp/sl-vue.pid

# 场景 C：B + Java 网关
docker compose up -d postgres redis orchestrator worker gateway
docker compose exec -T orchestrator alembic upgrade head
(cd sl-vue && npm install && nohup npm run dev > /tmp/sl-vue.log 2>&1 &)
echo $! > /tmp/sl-vue.pid
```

### 4.2 停止（Bash / Zsh）

```bash
# 停 Docker 服务
docker compose stop

# 关掉前端
if [ -f /tmp/sl-vue.pid ]; then
    kill $(cat /tmp/sl-vue.pid) 2>/dev/null
    rm /tmp/sl-vue.pid
else
    # 兜底：按端口找
    lsof -ti:5173 | xargs -r kill -9
fi
```

### 4.3 启动健康验证（任意平台）

```bash
# Python API
curl -s http://localhost:8000/health
# 期望：{"status":"ok","version":"...","redis":"connected","database":"configured"}

# Java Gateway（若起了）
curl -s http://localhost:8080/actuator/health
# 期望：{"status":"UP",...}

# 前端
# 浏览器打开 http://localhost:5173
```

## 五、单服务命令速查

```bash
# 查看容器状态
docker compose ps

# 查看所有容器资源消耗（CPU/内存）
docker stats --no-stream

# 启停单个服务
docker compose up -d <service>
docker compose stop <service>
docker compose restart <service>     # 只重启，不重建镜像
docker compose up -d --force-recreate --no-build <service>   # 用现有镜像重建容器

# 进入容器
docker compose exec <service> bash
docker compose exec -T postgres psql -U postgres -d myaivideos
docker compose exec -T redis redis-cli

# 改 Python 代码后必须 build（restart 不行！）
docker compose build orchestrator
docker compose up -d --force-recreate --no-build orchestrator worker

# 改前端代码：dev 模式自带 HMR，保存即生效
# build 生产包：
cd sl-vue
npm run build       # 输出到 dist/
npm run preview     # 预览 build 结果（端口 4173）
```

## 六、日志查看与排查

### 6.1 实时跟踪日志

```bash
# 看所有服务
docker compose logs -f

# 单个服务
docker compose logs -f orchestrator
docker compose logs -f worker
docker compose logs -f gateway

# 多个服务同时跟（错误关联）
docker compose logs -f orchestrator worker
```

### 6.2 历史日志（文件归档）

应用内置滚动日志归档，写入 `/app/data/logs/`（容器内）/ `./data/logs/`（宿主机）。即使容器重建，历史日志仍在。

```bash
# 看最新 orchestrator 日志
tail -f data/logs/orchestrator.log

# 看 worker
tail -f data/logs/worker.log

# 看历史滚动备份
ls -lh data/logs/
# orchestrator.log         # 最新
# orchestrator.log.1       # 前一个滚动
# orchestrator.log.2       # ...
```

### 6.3 跨服务追踪一条请求

每条日志带 `[trace=xxxxxxxx]`，跨 Java / Python 同一请求共用同一 trace_id：

```bash
# Linux / macOS / Git Bash
grep "trace=b09df48c0b714426" data/logs/*.log
docker compose logs orchestrator worker gateway 2>&1 | grep "trace=b09df48c0b714426"

# Windows PowerShell
Select-String -Path data\logs\*.log -Pattern "trace=b09df48c0b714426"
```

### 6.4 只看错误

```bash
# 实时
docker compose logs -f orchestrator | grep -E "ERROR|Exception|Traceback"

# 最近 5 分钟
docker compose logs --since 5m orchestrator gateway worker | grep -E "ERROR|Traceback"

# 文件归档里的错误
grep -E "\[ERROR\]" data/logs/orchestrator.log | tail -50
```

### 6.5 数据库检查

```bash
# 进 psql
docker compose exec -T postgres psql -U postgres -d myaivideos

# 单条 SQL
docker compose exec -T postgres psql -U postgres -d myaivideos -c "
SELECT count(*) FROM plans;
SELECT count(*) FROM video_records;
"

# 当日 LLM 调用统计
docker compose exec -T postgres psql -U postgres -d myaivideos -c "
SELECT tenant_id, model, total_calls, total_cost_cny, fallback_calls
  FROM billing_daily
 WHERE dt = CURRENT_DATE;
"

# 最近一次错误的 LLM 调用
docker compose exec -T postgres psql -U postgres -d myaivideos -c "
SELECT id, trace_id, provider_final, latency_ms, created_at
  FROM llm_calls
 ORDER BY created_at DESC LIMIT 10;
"
```

### 6.6 Redis 队列检查

```bash
# 进 redis-cli
docker compose exec -T redis redis-cli

# 看队列长度
docker compose exec -T redis redis-cli LLEN arq:queue

# 看所有 keys（小心：生产慎用）
docker compose exec -T redis redis-cli --scan | head -20
```

### 6.7 健康探针

```bash
# Python API 详细状态
curl -s http://localhost:8000/health | jq

# Java Gateway 各组件
curl -s http://localhost:8080/actuator/health | jq '.components'

# 看 Prometheus 指标
curl -s http://localhost:8080/actuator/prometheus | grep -E "llm_|circuit_|gateway_"
```

## 七、数据库迁移

```bash
# 升级到最新
docker compose exec -T orchestrator alembic upgrade head

# 看当前版本
docker compose exec -T orchestrator alembic current

# 全新库：表已经被 init_db 建好 → 撞 "relation already exists"
# 用 stamp 标记为已应用（不再跑 DDL）：
docker compose exec -T orchestrator alembic stamp head

# 回滚一步
docker compose exec -T orchestrator alembic downgrade -1

# 创建新迁移
docker compose exec -T orchestrator alembic revision -m "your message"
```

## 八、常见故障速查

| 现象 | 原因 | 处理 |
|---|---|---|
| `port 5173 already in use` | 上次前端没停干净 | Win: `Get-NetTCPConnection -LocalPort 5173`；Mac/Linux: `lsof -ti:5173 \| xargs kill` |
| `port 8000 / 8080 already in use` | Docker 上次没停 | `docker compose stop <service>`；或改 compose 端口 |
| `Cannot connect to the Docker daemon` | Docker Desktop 没启动 | 打开 Docker Desktop |
| `relation "xxx" already exists` | 全新库，`create_all` 与 alembic 撞 | `docker compose exec -T orchestrator alembic stamp head` |
| 改了 Python 代码部署后仍是旧逻辑 | 用了 `restart` 没重 build | `docker compose build <service> && docker compose up -d --force-recreate --no-build <service>` |
| Gateway 启动报 `IllegalStateException: dev/example value` | `.env` 缺 `GATEWAY_AUTH_JWT_SECRET` 或值含弱模式 | `openssl rand -base64 32` 后写入 `.env` |
| Gateway 401 `JWT invalid` | Token 过期 / 签名 secret 不匹配 | 重签 token；`docker compose logs gateway \| grep "jwt parse"` |
| 前端 5xx / 500 | Python orchestrator 没起 / 崩了 | `docker compose logs orchestrator --tail 50`；查 `/app/data/logs/orchestrator.log` |
| `npm install` 慢 | 默认源走 npmjs.com | `npm config set registry https://registry.npmmirror.com` |
| `pip install` 慢 / 超时 | 默认 PyPI 慢 | Dockerfile 已配清华源 + timeout 重试 |
| Kling / TTS 调用超时 | 上游网络抖动 / 限流 | 看 `kling_circuit_breaker_state` 指标；OPEN 即熔断已生效保护 |
| 日志找不到 trace_id | 老版本未走 `setup_logging` | 重 build orchestrator + worker 镜像 |
| 容器自动重启循环 | 配置错误 / 启动失败 | `docker compose logs <service>` 看 stack trace |
| 磁盘满 | Docker 镜像 / 旧片段堆积 | 见 [九、彻底重置](#九彻底重置慎用) |

## 九、彻底重置（慎用）

⚠️ **删除所有本地数据，包括数据库、对象存储、缓存、Docker 镜像**。仅用于新电脑彻底重装或排障最后兜底。

```bash
# 停容器 + 删 volume（DB 数据全没！）
docker compose down -v

# 删项目镜像
docker rmi ai-platform-app:latest ai-platform-gateway:latest -f

# 清 Docker build cache
docker builder prune -af

# 删前端依赖与产物
# Linux/macOS:
rm -rf sl-vue/node_modules sl-vue/dist
# Windows PowerShell:
Remove-Item -Recurse -Force sl-vue\node_modules, sl-vue\dist

# 删历史日志归档
# Linux/macOS:
rm -rf data/logs/
# Windows PowerShell:
Remove-Item -Recurse -Force data\logs

# 然后重新执行 [三/四] 的启动流程
```

## 十、维护说明

- 端口冲突或新增服务 → 改 `docker-compose.yaml` 后同步更新本文 §1.3
- 新增环境变量 → 同步加到 `.env.example` 和本文 §1.2
- 新增启停脚本 → 放在 `scripts/` 下，在 §3.3 / §4.3 引用
- 任何一次"踩坑修复"建议在 §8 留一条速查
